## 3. セキュリティ設計

### なぜ安全なのか？

```
❌ 危険な設計（やってはいけない）:
  フロントエンドJS内に:
    const SHEET_ID = '1BxiMVs0XRA5...';  // 丸見え！
    const API_KEY  = 'AIzaSyD...';       // 丸見え！

✅ 今回の安全な設計:
  フロントエンドJS内には:
    const GAS_URL = 'https://script.google.com/macros/s/.../exec';
    // URLのみ。スプレッドシートIDもAPIキーも含まない。
    
  GAS (サーバー側) の中にのみ:
    const SPREADSHEET_ID = '1BxiMVs0XRA5...';  // ユーザーからは見えない
```

### セキュリティの注意点

1. **GAS_URLはURLとして公開される** → URLを知っていれば誰でもAPIを叩ける
2. **会員番号のみの認証** → セキュリティ強度は高くない
3. **本格運用時の推奨強化策**:
   - 会員番号 + PINコード（4桁）の2要素認証
   - レートリミット（短時間の大量リクエスト拒否）
   - IPアドレスログ記録
   - セッションタイムアウト

## 4. 導入手順（ステップ・バイ・ステップ）

### STEP 1: Googleスプレッドシートを作成

1. https://sheets.google.com にアクセス
2. 「新しいスプレッドシート」を作成
3. シート名を「会員データ」に変更
4. URLから **スプレッドシートID** をコピー
   ```
   https://docs.google.com/spreadsheets/d/【ここがID】/edit
   ```

### STEP 2: Google Apps Script を設定

1. スプレッドシートのメニュー「拡張機能」→「Apps Script」を開く
2. デフォルトのコードを全て削除
3. 提供した `コード.gs` の内容を全てペースト
4. 以下の2行を自分の情報に書き換える:
   ```javascript
   const SPREADSHEET_ID = '★コピーしたスプレッドシートID★';
   const ADMIN_PASSWORD = '★任意の管理者パスワード★';
   ```
5. 保存（Ctrl+S）

### STEP 3: 初期データをセットアップ

1. GASエディタで関数 `setupInitialData` を選択
2. 「実行」ボタンをクリック
3. 権限承認を求められたら「承認する」を押す
4. スプレッドシートにサンプルデータが追加されたことを確認

### STEP 4: GAS を Web アプリとしてデプロイ

1. GASエディタ右上「デプロイ」→「新しいデプロイ」
2. 種類: 「ウェブアプリ」
3. 設定:
   - 説明: `Retro Arcade Hub API v1`
   - 次のユーザーとして実行: **「自分」**
   - アクセスできるユーザー: **「全員」**（匿名を含む）
4. 「デプロイ」をクリック
5. 表示された **ウェブアプリURL** をコピー
   ```
   https://script.google.com/macros/s/【デプロイID】/exec
   ```

### STEP 5: フロントエンドにURLを設定

`index.html` の以下の行を書き換える:
```javascript
// 変更前
const GAS_URL = 'https://script.google.com/macros/s/YOUR_DEPLOYMENT_ID/exec';

// 変更後
const GAS_URL = 'https://script.google.com/macros/s/【実際のデプロイID】/exec';
```

### STEP 6: 既存サイトに統合

**方法A: HTMLに直接追加（推奨）**
既存の `index.html` の `<head>` 内に `<style>` を追加し、
`<body>` の先頭に `#login-overlay`, `#member-hud` 等のHTMLを追加する。
`</body>` の直前に `<script>` タグを追加する。

**方法B: JavaScript を分離**
1. CSSを `member-system.css` として保存
2. JSを `member-system.js` として保存
3. `index.html` でそれぞれ読み込む:
   ```html
   <link rel="stylesheet" href="member-system.css">
   <script src="member-system.js"></script>
   ```

### STEP 7: GitHub Pages にデプロイ

```bash
git add .
git commit -m "Add member management system"
git push origin main
```

### STEP 8: 動作確認

1. サイトにアクセス → ログイン画面が表示されることを確認
2. テスト会員番号 `M001` でログイン → クレジット表示確認
3. 管理者パネルで会員追加・クレジット操作を確認

## 5. ゲームリンクへの統合方法

既存のゲームリンク（PLAY GAMEボタン）をクリックした際にクレジットを消費させる例:

```javascript
// 既存のゲームリンクを上書き
document.querySelectorAll('.play-game-btn').forEach(btn => {
  btn.addEventListener('click', async (e) => {
    e.preventDefault();
    const gameName = btn.dataset.game || 'ゲーム';
    const gameUrl  = btn.href;
    
    // クレジット1枚を消費
    const ok = await useCredit(gameName, 1);
    if (ok) {
      window.location.href = gameUrl; // ゲームページへ移動
    }
  });
});
```

## 6. よくあるトラブル

| 症状 | 原因 | 対処 |
|------|------|------|
| 「通信エラー」が出る | GAS_URLが間違い / GASがデプロイされていない | URLを再確認、デプロイし直す |
| ログインできない | シートに会員番号が存在しない | スプレッドシートで会員データを確認 |
| CORSエラー | GASのアクセス設定が「全員」になっていない | デプロイ設定を再確認 |
| 管理者ログインできない | パスワードが違う / コード.gsのパスワードと不一致 | コード.gsの `ADMIN_PASSWORD` を確認 |
| GASを更新したのに反映されない | 再デプロイが必要 | 「新しいデプロイ」or「バージョンを管理」で更新 |
